1.1 用户配置文件

  • 存储用户密码哈希值
  • 只有root用户可访问
  • 包含密码过期时间等安全信息

1.2 用户管理命令

useradd - 添加用户

常用选项：

  • -c：用户备注信息
  • -d：指定家目录路径
  • -g：指定主组
  • -G：指定附加组
  • -s：指定登录shell
  • -m：自动创建家目录(从/etc/skel复制模板)

usermod - 修改用户

常用选项：

  • -c：备注:指定备注
  • -d：path:home目录
  • -g：组名/GID:所属主组
  • -G：组名/GID:其他所属组
  • -s：path: login shell
  • -L：锁定用户密码
  • -U：解锁用户密码

userdel - 删除用户

passwd - 密码管理

3. 系统日志管理

3.1 syslog 配置

配置文件：/etc/syslog.conf

格式：

facility(设施)类型：

  • auth/authpriv：认证相关
  • cron：定时任务
  • daemon：守护进程
  • kern：内核消息
  • mail：邮件系统
  • syslog：syslog自身
  • local0-local7：自定义设施

priority(优先级)：

  • emerg：紧急情况
  • alert：需要立即处理
  • crit：严重错误
  • err：一般错误
  • warning：警告
  • notice：重要通知
  • info：一般信息
  • debug：调试信息

输出目标：

  • 文件名：如/var/log/messages
  • @host：远程syslog服务器
  • 用户名：发送到用户终端
  • *：所有用户终端

6.4 访问控制

cron 访问控制

  1. 如存在/etc/cron.allow，仅允许文件内用户
  2. 如无allow文件，检查/etc/cron.deny，拒绝文件内用户
  3. 如两者都不存在，默认行为取决于系统配置

at 访问控制

  1. 如存在/etc/at.allow，仅允许文件内用户
  2. 如无allow文件，检查/etc/at.deny，拒绝文件内用户
  3. 如两者都不存在，默认只有root可用
